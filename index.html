🎯 O Que é o Nexiflow Approvals?

Pense no Nexiflow como um "gerente de projetos imobiliários" que:

  1. Recebe as informações do seu projeto (via formulário online)
  2. Avalia se o projeto é viável (usando dados do IBGE + suas respostas)
  3. Aprova ou rejeita o projeto
  4. Guia você através das próximas etapas

É como um processo de análise bancária, mas para projetos imobiliários.

📋 Exemplo Real: O Projeto do João

Vamos acompanhar João que quer lançar um novo loteamento em São Paulo

Fase 1️⃣: João Preenche o Formulário

João acessa o site e escolhe:

Tipo: Lançamento
Subtipo: Loteamento (Parcelamento de Solo)

Ele preenche categorias:

Categoria Campos Exemplo
Informações Gerais Nome: "Loteamento Vila Nova" | Email: contato@vilanova.com.br
Localização Estado: São Paulo | Cidade: Osasco | CEP: 06085-100
Dados Demográficos População: 720 mil | PIB per capita: R$ 32.000 | IDH: 0,789
Características Tipo: Casa | Metragem: 300-500m² | 80 lotes | VGV: R$ 8M
Mercado Preço m²: R$ 6.500 | Demanda: Alta | Comissão: 5%
Documentos KMZ, Plantas, Alvará, etc

✅ O que Acontece Depois?

  1. Sistema valida todos os campos obrigatórios
  2. Sistema calcula um SCORE usando dados do IBGE
  3. Gera número: NXM-2026-ABC12345
  4. Email enviado: "Seu projeto foi registrado. Protocolo: NXM-2026-ABC12345"

🔍 Fase 2: Revisor Analisa o Projeto

Um especialista em projetos imobiliários (chamado "Revisor") acessa a plataforma e vê o projeto de João.

O revisor pode:

✅ APROVAR
Significa: "Seu projeto é viável! Vamos adiante."

Email: "Projeto aprovado! Link para próxima etapa: [URL com token]"
❌ REJEITAR
Significa: "Projeto não se enquadra."

Email: "Lamentamos, projeto rejeitado por: [razão]"
🔄 PEDIR CORREÇÕES
Significa: "Seu projeto precisa de mais informações."

Email: "Alguns campos precisam ser corrigidos. Acesse: [URL]"
João volta e refaz os campos marcados em vermelho

🎁 Fase 3 & 4: Próximas Etapas e Proposta

SE João escolheu "Lançamento" ou "Remanescente":

→ Vai direto para FASE 4 (Proposta)

SE João escolheu "Análise de Áreas":

→ Recebe link para preencher formulário complementar com ~30 perguntas adicionais

FASE 4: Proposta Customizada

Depois de tudo aprovado, o revisor cria uma PROPOSTA com:

  • ✓ Análise completa do mercado
  • ✓ Estimativa de viabilidade econômica
  • ✓ Recomendações de posicionamento
  • ✓ Próximos passos sugeridos

João recebe email com link único para baixar o PDF.

🚀 Como Usar o Sistema (Passo a Passo)

1️⃣ Enviar um Novo Projeto

Passo 1: Acessar a Plataforma

  • Abra: http://localhost:8081
  • Clique em "Novo Projeto"

Passo 2: Escolher o Tipo de Projeto

📊 Análise de Áreas - Quando você tem um terreno bruto
🏢 Lançamento - Quando você está lançando um novo imóvel
📈 Remanescente - Quando seu projeto antigo ainda tem unidades
🏗️ Análise Áreas/Incorporador - Quando você é incorporador com terreno

Passo 3: Preencher o Formulário

  • ⭐ Campos com asterisco (*) são obrigatórios
  • 💡 Campos com "?" têm ajuda extra ao passar mouse
  • 🔴 Se ficar vermelho, revise antes de continuar
  • ➡️ Use "Próximo" para avançar
  • ⬅️ Use "Anterior" para voltar

Passo 4: Dados do Responsável

Na primeira tela, preencha:

  • Seu Nome Completo
  • Seu Email
  • Seu Telefone

Esses dados aparecem em todo email enviado.

Passo 5: Submeter

Clique em "Enviar Projeto"

Sistema vai:

  1. Validar todos os campos
  2. Calcular o score
  3. Gerar protocolo
  4. Enviar email

2️⃣ Rastrear Seu Projeto

Opção A: Via Protocolo

  1. Acesse a plataforma
  2. Clique em "Rastrear Projeto"
  3. Copie/cole seu protocolo: NXM-2026-ABC12345
  4. Clique em "Buscar"

Opção B: Via Email

  1. Abra o email que recebeu
  2. Clique no link de rastreamento
  3. Você é levado direto ao seu projeto

Você verá:

  • Status atual (em análise / aprovado / rejeitado)
  • Data de envio
  • Timeline de eventos

3️⃣ Se Seu Projeto Precisa de Correções

Se o revisor respondeu "Aguardando Correções":

  1. Email recebido com link de acesso
  2. Clique no link → você volta ao formulário
  3. Campos vermelhos = o que precisa corrigir
  4. Sistema navega automaticamente para o 1º erro
  5. Corrija os campos indicados
  6. Clique "Reenviar"
✅ O sistema apaga a rejeição anterior e trata como novo envio

4️⃣ Se Seu Projeto Foi Aprovado

Você receberá email dizendo: "Seu projeto foi aprovado! 🎉"

SE é Lançamento/Remanescente:

→ Aguarde a proposta estar pronta
→ Email com link para baixar proposta PDF

SE é Análise de Áreas:

→ Email com link para formulário complementar
→ Isso é a FASE 3 (tem mais ~30 perguntas)
→ Preenchida a Fase 3, aí sim vem a proposta

5️⃣ Acessar a Proposta

Você recebeu email com:

Assunto: Sua Proposta Está Pronta!

Link de acesso:
http://localhost:8081/proposta/123abc?token=xyz789

Clique no link (vale copiar/colar também):

  • ✓ Sem necessidade de login
  • ✓ Link é único e seguro
  • ✓ Abre em PDF no navegador
  • ✓ Pode baixar/imprimir

📞 Dúvidas Frequentes

P: Posso editar meu projeto depois de enviar?

R: Apenas se o revisor pedir correções. Caso contrário, precisa enviar um novo projeto.

P: Por quanto tempo meu link de acesso funciona?

R: O link nunca expira naturalmente. Um admin pode revogá-lo manualmente se necessário.

P: Posso compartilhar meu protocolo com alguém?

R: Sim! Protocolo é público. Qualquer um pode rastrear o status. Mas o acesso ao formulário/proposta requer o link enviado por email (seguro).

P: Quanto tempo leva para o revisor responder?

R: Depende da SLA do cliente. O sistema registra tudo (data/hora) para auditoria.

3. Tipos de Projeto e Suas Perguntas

🎯 LANÇAMENTO (lancamento)

  • Fase: Apenas Phase 1
  • Total de perguntas: ~50
  • Próximo passo: Phase 4 (Proposta)

📈 REMANESCENTE (remanescente)

  • Fase: Apenas Phase 1
  • Total de perguntas: ~53 (maioria igual a lançamento + campos específicos)
  • Campos extras: Quantas unidades vendendo? Há quanto sem vender? GERIC?
  • Próximo passo: Phase 4 (Proposta)

🏗️ ANÁLISE DE ÁREAS (area_bruta)

  • Fase 1: ~80 perguntas em 12 categorias
  • Fase 3: ~35 perguntas adicionais (se aprovado)
  • Próximo passo: Phase 3 → Phase 4

📋 Phase 3 (Apenas para Análise de Áreas)

Depende do study_type:

  • Incorporação: ~15 perguntas (Detalhes, Mercado, Projeto Aprovado, Negociação)
  • Loteamento: ~45 perguntas (+ 19 documentos técnicos massivos)
  • Multipropriedade: ~20 perguntas (+ específicas multipropriedade)

4. Integração IBGE (Scoring)

Cliente preenche:
- Estado (UF)
- Cidade
- PIB per capita
- População
        │
        ▼
Sistema calcula SCORE:
├─ Valida dados
├─ Checa IBGE_CACHE (24h)
├─ Se cache válida: usa dados cachados
├─ Se não: chama API IBGE externa
└─ Armazena em projects.internal_score

Score é:
✓ SALVO no banco (JSONB)
✗ OCULTO do cliente
✓ VISÍVEL só para reviewer=true

5. Sistema de Emails (SMTP Direto)

Sistema envia emails diretos via SMTP para eventos:

Evento Quando Email Assunto
Project Submitted Cliente envia Phase 1 "Seu projeto foi recebido"
Project Approved Revisor clica APROVAR "Seu projeto foi aprovado"
Project Rejected Revisor clica REJEITAR "Projeto rejeitado"
Revision Requested Revisor clica PEDIR CORREÇÕES "Seu projeto precisa ajustes"
Resubmission Cliente resubmete após corr. "Resubmissão recebida"
Proposal Sent Admin envia proposta "Sua proposta está pronta"

6. Segurança: Tokens e Acesso

Tipos de Token:

access_token
• Phase 3 + Proposta
• UUID aleatório
• Enviado por email
• Nunca caduca
• Compartilhável (público)
auth_token (JWT)
• Admin/Reviewer
• Expiração: 24h
• Armazenado em localStorage
• Enviado em Authorization header
• Com role (admin/reviewer/client)

Regras de Acesso:

  • ✓ Cliente vê APENAS seu projeto (via token/email)
  • ✓ Reviewer vê TODOS (autenticado)
  • ✓ Admin vê TUDO + pode deletar
  • ✗ Ninguém vê score sem ?reviewer=true
  • ✓ Proposta = APENAS email/token, nunca login
