🎯 O Que é o Nexiflow Approvals?

Pense no Nexiflow como um "gerente de projetos imobiliários" que:

1. Recebe as informações do seu projeto ▼

O sistema coleta dados através de um formulário online estruturado em categorias. Tudo é validado em tempo real para garantir que as informações sejam completas e precisas.

2. Avalia se o projeto é viável ▼

Integração com dados do IBGE (população, PIB) combinados com suas respostas para gerar um score de viabilidade automático. É como um relatório preliminar objetiva.

3. Aprova ou rejeita o projeto ▼

Um especialista revisor analisa cada projeto e pode aprovar, rejeitar ou solicitar correções. Cada ação é registrada no histórico para auditoria completa.

4. Guia você através das próximas etapas ▼

Dependendo do tipo de projeto, o sistema gerencia fases adicionais: formulários complementares, análises de mercado e propostas customizadas.

👤 Como Usar o Sistema (Passo a Passo)

Guia operacional: instruções detalhadas para cada ação.

1️⃣ Enviar um Novo Projeto ▼

Passo 1: Acessar a Plataforma

  • Abra: http://localhost:8081
  • Clique em "Novo Projeto" ou "Enviar Análise"

Passo 2: Escolher o Tipo de Projeto

Tipo Quando Usar
Análise de Áreas Terreno bruto → viável para construir?
Lançamento LANÇANDO novo imóvel/condomínio
Remanescente Projeto antigo com unidades disponíveis
Incorporador Incorporador com terreno disponível

Passo 3: Preencher o Formulário

O formulário é dividido em categorias (abas).

  • ⭐ Campos com asterisco (*) são obrigatórios
  • 💡 Campos com "?" têm ajuda ao passar mouse
  • 🔴 Se ficar vermelho, revise antes de continuar
  • ➡️ Clique "Próximo" para avançar
  • ⬅️ Clique "Anterior" para voltar

Passo 4: Dados do Responsável

Na primeira tela, preencha:

  • Seu Nome Completo
  • Seu Email (aparece em todos emails)
  • Seu Telefone

Passo 5: Submeter

Clique em "Enviar Projeto" no final. O sistema vai:

  1. Validar todos os campos
  2. Calcular o score
  3. Gerar protocolo
  4. Enviar email de confirmação
2️⃣ Rastrear Seu Projeto ▼

Opção A: Via Protocolo

  1. Acesse a plataforma
  2. Clique em "Rastrear Projeto"
  3. Copie/cole seu protocolo: NXM-2026-ABC12345
  4. Clique em "Buscar"

Opção B: Via Email

  1. Abra o email que recebeu
  2. Clique no link de rastreamento
  3. Você é levado direto ao seu projeto

Você verá: Status atual, data de envio, timeline de eventos.

3️⃣ Se Seu Projeto Precisa de Correções ▼

Se o revisor respondeu "Aguardando Correções":

  1. Email recebido com link de acesso
  2. Clique no link → volta ao formulário
  3. Campos vermelhos = o que precisa corrigir
  4. Sistema navega automaticamente para o 1º erro
  5. Corrija os campos indicados
  6. Clique "Reenviar"

✅ O sistema apaga a rejeição anterior e trata como novo envio.

4️⃣ Se Seu Projeto Foi Aprovado ▼

Você receberá email dizendo: "Seu projeto foi aprovado!"

SE é Lançamento/Remanescente:

  • Aguarde a proposta estar pronta
  • Email com link para baixar proposta PDF

SE é Análise de Áreas:

  • Email com link para formulário complementar
  • Isso é a FASE 3 (tem mais ~35 perguntas)
  • Após completar, recebe a proposta
5️⃣ Acessar a Proposta ▼

Você recebeu email com:

Assunto: Sua Proposta Está Pronta! Link de acesso: http://localhost:8081/proposta/123abc?token=xyz789

Clique no link:

  • ✓ Sem necessidade de login
  • ✓ Link é único e seguro
  • ✓ Abre em PDF no navegador
  • ✓ Pode baixar/imprimir

👨‍💻 Arquitetura e Fluxos Técnicos

Documentação técnica: fluxos, banco de dados, integrações e modelos de dados.

1. Modelo de Dados ▼

Tabela Principal: PROJECTS

  • Informações Básicas: user_name, user_email, user_phone
  • Localização: uf, city
  • Tipo: entry_type (lancamento/remanescente/area_bruta), study_type
  • Status: status, current_phase
  • Respostas: phase1_answers (JSONB), phase3_answers (JSONB)
  • Scoring: internal_score (JSONB)
  • Rastreamento: protocol_number, access_token, created_at, updated_at
  • Revisões: revision_requested (JSONB)
  • Proposta: proposal (JSONB), proposal_token
  • Soft Delete: deleted_at

Tabelas Relacionadas:

  • REVIEW_ACTIONS: Auditoria de cada ação do revisor
  • REVIEW_COMMENTS: Comentários campo por campo
  • IBGE_CACHE: Cache de dados IBGE (24h)
  • USERS: Admin, reviewer, client (JWT auth)
2. Fluxo de Status (State Machine) ▼
┌─────────────────────────────────┐ │ NOVO PROJETO ENVIADO │ │ Status: pendente_analise │ │ Phase: 1 │ └──────────────┬──────────────────┘ │ ┌──────────────▼──────────────────┐ │ REVISOR ANALISA │ └──┬──────────────┬──┬────────────┘ │ │ │ ┌──▼──┐ ┌────▼─▼───┐ │APROV│ │ REJEITA │ │ADOR │ └──────────┘ └─────┤ │ ┌─────▼──────────────────────────┐ │ TIPO DEFINE PRÓXIMO PASSO │ ├────────────────────────────────┤ │ SE Lançamento/Remanescente: │ │ → FASE 4 (Proposta) │ │ Status: aguardando_proposta │ │ │ │ SE Análise de Áreas: │ │ → FASE 3 (Complementar) │ │ Status: fase_complementar │ │ Phase: 3 │ └────────────────────┬───────────┘ │ ┌─────────▼────────┐ │ PROPOSTA ENVIADA │ │ Status: final │ └──────────────────┘
3. Fluxo de Revisão ▼

O revisor tem 4 ações disponíveis:

Ação Resultado Email Disparado
APROVAR Status → fase_complementar (se areas) ou aguardando_proposta Confirmação com próximo link
REJEITAR Status → reprovado (FINAL), motivo salvo Motivo da rejeição
PEDIR CORREÇÕES Status → aguardando_correcao, campos marcados Link para corrigir com campos em destaque
COMENTAR Salvo em review_comments, visível no histórico Não dispara email automático
4. Segurança: Tokens e Acesso ▼
Token Tipo Uso Expira?
access_token UUID aleatório Phase 3 + Proposta Nunca (admin revoga)
proposal_token UUID aleatório Apenas Proposta Nunca
auth_token JWT com expiração Admin/Reviewer APIs Sim (24h)

Regras de Acesso:

  • Cliente vê APENAS seu projeto (via token/email)
  • Reviewer vê TODOS (autenticado)
  • Admin vê tudo + pode deletar
  • Ninguém vê score sem ?reviewer=true
  • Proposta = APENAS email/token, nunca login
5. Eventos de Email (SMTP) ▼
Evento Quando Assunto
Project Submitted Cliente envia Phase 1 "Seu projeto foi recebido"
Project Approved Revisor clica APROVAR "Seu projeto foi aprovado"
Project Rejected Revisor clica REJEITAR "Projeto rejeitado"
Revision Requested Revisor clica PEDIR CORR. "Seu projeto precisa ajustes"
Resubmission Received Cliente resubmete após corr "Resubmissão recebida"
Proposal Sent Admin envia proposta "Sua proposta está pronta"

📋 Estrutura dos Formulários

Detalhes sobre quantas perguntas, categorias e campos especiais por tipo de projeto.

A. LANÇAMENTO (lancamento) ▼

Apenas Phase 1 ~50 perguntas 9 categorias

Campos especiais:

  • Histórico construtora
  • Dados de mercado do empreendimento
  • Status da obra
  • VGV (Valor Geral de Vendas)
  • Número de unidades vendidas

Próximo passo: Phase 4 (Proposta direto)

B. REMANESCENTE (remanescente) ▼

Apenas Phase 1 ~53 perguntas 9 categorias

Campos especiais (único de remanescente):

  • "Quantas unidades eram no total?"
  • "Está vendendo quantas por mês?"
  • "Há quantos meses sem vender?"
  • "GERIC? Se não, por quê?"
  • "Por que não está vendendo?" (análise qualitativa)

Próximo passo: Phase 4 (Proposta direto)

C. ANÁLISE DE ÁREAS (area_bruta / incorporador_terreno) ▼

Phase 1: ~80 perguntas Phase 3: ~35 perguntas 12 categorias (Phase 1)

Phase 1 - Estrutura (12 categorias):

Categoria Perguntas Tópico
1. Informações Gerais 3 Nome, tipo estudo, email
2. Localização 2 Estado, cidade
3. Dados Demográficos 8 População, PIB, IDH, atividade
4. Características do Terreno 11 Tamanho, topografia, zoneamento, etc
5. Plano Diretor 4 Coef aproveitamento, taxa ocupação
6. Infraestrutura 4 Água, esgoto, energia, asfalto
7. Dados de Mercado 9 Preço, oferta/demanda, VGV
8. Ambiental 2 Passivo, APP
9. Negociação 12 Permuta, jurídico, comissão
10. Multipropriedade 12 Só se study_type = Multipropriedade
11. Documentos 6 KMZ, matrícula, planialtimétrico
12. Observações 1 Comentários gerais

Phase 3 - Complementar (depende de study_type):

Se aprovado em Phase 1, o cliente acessa Phase 3 com formulário específico:

  • INCORPORAÇÃO: Detalhes área, mercado, projeto aprovado, proprietário
  • LOTEAMENTO: MESMOS tópicos + 19 documentos técnicos (licenças, projetos, etc)
  • MULTIPROPRIEDADE: Multipropriedade especial (intercambiadora, parcerias)

❓ Perguntas Frequentes (FAQ)

P: Como o scoring funciona exatamente? ▼

O sistema usa população + PIB per capita do IBGE + dados que você preenche (topografia, acessos, infraestrutura, mercado) para gerar uma pontuação de 0-100. Isso ajuda o revisor a ter uma primeira impressão objetiva do projeto.

P: Posso editar meu projeto depois de enviar? ▼

Apenas se o revisor pedir correções. Caso contrário, precisa enviar um novo projeto.

P: Por quanto tempo meu link de acesso funciona? ▼

O link nunca expira naturalmente. Um admin pode revogá-lo manualmente se necessário.

P: O revisor vê meus comentários (observações)? ▼

Sim, tudo é registrado no sistema. É um diário de bordo do projeto completo, incluindo todas as observações e comentários adicionados.

P: Posso compartilhar meu protocolo com alguém? ▼

Sim! Protocolo é público. Qualquer um pode rastrear o status. Mas o acesso ao formulário/proposta requer o link enviado por email (seguro com token).

P: E se o revisor não responder? ▼

Depende da SLA (Service Level Agreement) do cliente. O sistema registra tudo (data/hora) para auditoria completa.
