📖 Guia do Cliente

✅ Como começar

Bem-vindo ao Nexiflow Approvals! Este guia explica como usar o sistema passo a passo.

1️⃣ Primeira Etapa: Preenchimento do Formulário

📝 Informações Básicas ▼

Na primeira fase, você precisará preencher:

  • Tipo de Entrada: Lançamento, Remanescente, Área Bruta ou Incorporador/Terreno
  • Tipo de Estudo: Comercial, Ambiental, Técnico ou Outro
  • Localização: Estado e Cidade do projeto
  • Dados Pessoais: Nome, Email e Telefone
  • Checklist: Confirme a disponibilidade de documentos

2️⃣ Segunda Etapa: Análise do Revisor

🔍 O que acontece após submissão ▼

Após enviar o formulário, um revisor analisará seu projeto:

  • Um número de protocolo será gerado (ex: NXM-2026-AB12)
  • Um email de confirmação será enviado
  • O revisor avaliará seus documentos em até 5 dias úteis
  • Você pode acompanhar o status em "Meu Protocolo"

3️⃣ Terceira Etapa: Formulário Complementar (quando necessário)

📋 Dados Adicionais ▼

Se seu projeto for aprovado, você receberá um email com um link para preencher dados complementares:

  • Detalhes técnicos do projeto
  • Informações financeiras
  • Upload de documentos adicionais
  • Cronograma de execução

⚠️ Importante: Este formulário só aparece se seu projeto foi aprovado na fase 1.

4️⃣ Quarta Etapa: Proposta do Projeto

📊 Visualizar a Proposta ▼

Após completar o formulário complementar, você poderá:

  • Acessar a proposta via link único enviado por email
  • Visualizar análise técnica do seu projeto
  • Ver recomendações dos revisores
  • Download de documentos em PDF

👨‍💻 Guia do Usuário (Reviewer/Admin)

⚠️ Acesso Restrito

Esta seção é para revisores e administradores do sistema. Requer autenticação.

🔐 Autenticação

🔑 Login e Permissões ▼

Para acessar o painel de revisão:

  • Acesse /admin/login
  • Use suas credenciais (email e senha)
  • Perfis disponíveis: Reviewer, Admin
  • Cada perfil tem permissões diferentes
💡 Dica

Se esqueceu a senha, clique em "Recuperar Senha" na página de login.

📊 Dashboard do Revisor

📈 Visualização de Projetos ▼

No dashboard, você verá:

  • Lista de Projetos: Todos os projetos submetidos
  • Status: Pendente, Em Análise, Aprovado, Rejeitado, Aguardando Correção
  • Filtros: Por data, tipo de entrada, status, etc.
  • Busca: Procure por protocolo ou nome do cliente
  • Score Automático: Avaliação numérica do projeto
✅ Score Automático

O sistema calcula automaticamente um score de 0-100 baseado em dados IBGE (população, PIB, etc.). Este é um indicador da viabilidade econômica do projeto.

✍️ Ações do Revisor

🎯 Aprovar, Rejeitar ou Solicitar Revisão ▼

Para cada projeto, você pode:

  • Aprovar: Projeto passa para próxima fase. Cliente recebe link para formulário complementar.
  • Rejeitar: Projeto é marcado como rejeitado. Cliente recebe notificação com motivo.
  • Solicitar Revisão: Projeto volta para o cliente com observações. Cliente pode reenviar.
  • Deixar Comentários: Adicione observações em campos específicos do formulário.
⚠️ Importante

Toda ação fica registrada no sistema com data, hora e quem realizou. Este histórico é visível para auditoria.

🔌 APIs Disponíveis

🚀 Endpoints Principais ▼

Base URL: http://localhost:3001/api

Projetos

POST /projects - Criar novo projeto GET /projects - Listar todos projetos GET /projects/:id - Obter detalhes do projeto PATCH /projects/:id - Atualizar projeto GET /projects/:id?reviewer=true - Obter com score (apenas reviewers) GET /projects/protocolo/:no - Buscar por número de protocolo GET /projects/token/:token - Buscar por access_token (públic)

Reviewer Actions

POST /projects/:id/approve - Aprovar projeto POST /projects/:id/reject - Rejeitar projeto POST /projects/:id/request-revision - Solicitar revisão POST /projects/:id/comments - Adicionar comentários GET /projects/:id/review-actions - Ver histórico de ações

Proposta (Fase 4)

POST /projects/:id/market-study - Salvar estudo de mercado POST /projects/:id/send-proposal - Enviar proposta GET /projects/:id/proposta?token=... - Visualizar proposta (público)

Autenticação

POST /auth/login - Fazer login POST /auth/logout - Fazer logout POST /auth/refresh - Renovar token JWT GET /auth/me - Obter dados do usuário autenticado

🗄️ Estrutura do Banco de Dados

📋 Tabelas Principais ▼

Tabela: projects

Colunas principais:

  • id (UUID) - ID único do projeto
  • user_name (varchar) - Nome do cliente
  • user_email (varchar) - Email do cliente
  • user_phone (varchar) - Telefone do cliente
  • protocol_number (varchar, UNIQUE) - Número gerado (NXM-YYYY-XXXXX)
  • status (enum) - pendente_analise, fase_complementar, rejeitado, aguardando_proposta, proposta_enviada, etc
  • current_phase (int) - 1, 2, 3, 4 (fases do fluxo)
  • phase1_answers (JSONB) - Respostas do formulário inicial
  • phase3_answers (JSONB) - Respostas do formulário complementar
  • internal_score (JSONB) - Score automático (0-100)
  • access_token (UUID) - Token para acesso Phase 3
  • revision_requested (JSONB) - Observações de revisão
  • market_study (JSONB) - Estudo de mercado
  • proposal (JSONB) - Conteúdo da proposta
  • proposal_token (UUID) - Token para visualizar proposta
  • created_at, updated_at (timestamp) - Datas de criação e atualização
  • deleted_at (timestamp) - Para soft delete

Tabela: review_actions

Registra cada ação do revisor:

  • id (UUID) - ID único da ação
  • project_id (UUID) - FK para projects
  • reviewer_id (UUID) - FK para users (revisor que agiu)
  • action (enum) - approved, rejected, revision_requested
  • reason (text) - Motivo/observação
  • created_at (timestamp) - Quando ocorreu

Tabela: review_comments

Comentários em campos específicos:

  • id (UUID) - ID único do comentário
  • project_id (UUID) - FK para projects
  • field_name (varchar) - Qual campo recebeu comentário
  • comment (text) - Texto do comentário
  • reviewer_id (UUID) - Quem comentou
  • created_at (timestamp) - Quando comentou

🔒 Score Automático

📊 Algoritmo de Pontuação ▼

O score é calculado automaticamente via IBGE API:

  • Dados utilizados: População da cidade, PIB per capita, IDH
  • Range: 0 a 100
  • Cache: Resultados cachados por 24h em ibge_cache table
  • Fallback: Se API falhar, permite preenchimento manual
  • Visibilidade: Só aparece para reviewers (?reviewer=true)
⚠️ Segurança

O score NUNCA é enviado para clientes. É calculado server-side e apenas retornado a revisores autenticados.

❓ Perguntas Frequentes

❓ Quanto tempo leva para obter resposta? ▼

Geralmente, a análise leva entre 3 a 5 dias úteis após a submissão. Você receberá atualizações por email.

❓ Posso editar meu formulário após envio? ▼

Você não pode editar diretamente. Se o revisor solicitar correções, você receberá um email com instruções de como reenviar o formulário corrigido.

❓ O que significa o "Score" mostrado no dashboard? ▼

O score (0-100) é uma avaliação automática baseada em dados da cidade (população, PIB, IDH). Não é uma aprovação ou rejeição - é apenas um indicador de viabilidade econômica que auxilia o revisor.

❓ Meu projeto foi rejeitado. Posso reenviar? ▼

Se rejeitado, você pode criar um novo projeto no sistema. Use o mesmo email, e um novo protocolo será gerado. O sistema rastreará os projetos anteriores.

❓ Como acompanho meu projeto? ▼

Você pode acompanhar usando o número de protocolo gerado após submissão. Acesse a página "Meu Protocolo" e insira o número. Você verá o status em tempo real.

❓ Quais tipos de projeto são aceitos? ▼

Tipos de Entrada:

  • Lançamento: Novo empreendimento imobiliário
  • Remanescente: Projeto residual/complementar
  • Área Bruta: Venda de área sem benfeitoria
  • Incorporador/Terreno: Empreendimento de renovação/desenvolvimento
❓ O que aconteça na Fase 3? ▼

Se seu projeto for aprovado na Fase 1, você receberá um link para preencher formulário complementar com dados técnicos e financeiros. Isto é obrigatório para projetos tipo "Área Bruta" e "Incorporador/Terreno", mas opcional para "Lançamento" e "Remanescente".

❓ Como recebo a proposta? ▼

Após completar o formulário complementar, a proposta será gerada pelo revisor. Você receberá um email com um link único para visualizar a proposta. Este link é pessoal e único para seu projeto.

❓ Preciso fazer login para acompanhar meu projeto? ▼

Não. Para clientes, não é necessário login. Você acompanha usando:
• Protocolo + Email para Fase 1
• Link de acesso (access_token) para Fase 3
• Link de proposta (proposal_token) para Fase 4
